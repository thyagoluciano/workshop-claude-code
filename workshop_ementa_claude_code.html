Workshop — Boas práticas no Claude Code

Ementa baseada na documentação oficial da Anthropic e práticas adotadas pela comunidade

MÓDULO 1 Introdução

O que é o Claude Code e como ele pensa

  • Loop agêntico: como o Claude explora, planeja e implementa
  • A constraint central: a context window e por que ela importa
  • Diferença entre chatbot e agente de código
  • Instalação, configuração e primeiros passos

MÓDULO 2 Fundamento

CLAUDE.md — O "cérebro permanente" do projeto

  • O que incluir (comandos bash, estilo de código, convenções de branch)
  • O que nunca incluir (o que o Claude já sabe)
  • Usando /init para gerar um arquivo base automaticamente
  • Mantendo o arquivo enxuto: a regra do "causaria erros se removido?"
  • Skills vs. CLAUDE.md: quando usar cada um

MÓDULO 3 Fundamento

O fluxo Explorar → Planejar → Implementar → Commit

  • Plan mode: separar pesquisa de execução
  • Como forçar o Claude a ler o código antes de escrever
  • Critérios de verificação: testes, screenshots, outputs esperados
  • Quando pular o planejamento (tarefas pequenas e escopo claro)
  • Commits descritivos e criação de PRs com o Claude

MÓDULO 4 Fundamento

Prompts eficazes e contexto rico

  • Escopo da tarefa: arquivo, cenário, preferências de teste
  • Referenciar padrões existentes no codebase com @
  • Passar URLs, imagens, logs via pipe (cat error.log | claude)
  • Descrever sintoma + localização + definição de "resolvido"
  • Quando prompts vagos são úteis (exploração aberta)

MÓDULO 5 Fundamento

Gestão da context window

  • Por que a performance degrada conforme o contexto enche
  • Usando /clear estrategicamente entre sessões
  • Leituras seletivas de arquivo em vez de exploração ampla
  • Documentação viva (dev docs, living plans) para reduzir tokens
  • Monitorando uso de contexto com status line customizada

MÓDULO 6 Avançado

MCP — Model Context Protocol

  • O que é MCP e como ampliar as capacidades do Claude Code
  • Servidores locais (stdio) vs. remotos (HTTP/SSE)
  • Configurando servidores MCP com .mcp.json
  • Allowlist de servidores: tratar MCP como dependência de software
  • Casos de uso: Git, banco de dados, APIs externas, automações

MÓDULO 7 Avançado

Hooks — automação e controle de qualidade

  • PreToolUse e PostToolUse: interceptar ações antes/depois
  • Stop hooks: rodar build, typecheck e testes ao fim de cada resposta
  • Hooks para bloquear commits com secrets expostos
  • Auto-formatação na escrita de arquivos
  • Hooks como quality gates: aprovação automática de operações seguras

MÓDULO 8 Avançado

Subagentes, sessões paralelas e Git Worktrees

  • Como o Claude delega tarefas a subagentes (Task tool)
  • Rodando sessões paralelas com isolamento via Git Worktrees
  • Escolha do modelo por tarefa: Opus, Sonnet, Haiku
  • Controle de subagentes com --disallowedTools
  • Padrões multi-agente: revisor, planejador, executor

MÓDULO 9 Segurança

Permissões, segurança e prevenção de riscos

  • Modos de permissão: plan, default, bypass — quando usar cada um
  • Regras de deny: bloquear acesso a .env, ~/.ssh/, secrets
  • Nunca rodar Claude Code como root
  • Prompt injection via README, comentários e respostas de MCP
  • CVEs conhecidos: lições de repositórios maliciosos e configs envenenadas
  • managed-settings.json para políticas de equipe/empresa

MÓDULO 10 Produção

Uso em equipes e ambientes de produção

  • Rotação de API keys e monitoramento de uso não autorizado
  • Retenção de transcrições e conformidade (LGPD / dados sensíveis)
  • Skills compartilhadas para times: slash commands padronizados
  • Gestão de custos: rate limits, seleção de modelo e uso em batch
  • Iteração contínua: refinando CLAUDE.md com base nos erros do Claude
