Modulo 4 . Topico 2

Referenciar Padroes Existentes com @

Em vez de descrever um padrao com palavras, aponte diretamente para o arquivo que ja implementa esse padrao. O operador @ no Claude Code faz com que o agente leia o conteudo real do arquivo, garantindo que ele entenda exatamente como seu codigo funciona.

O operador @ no Claude Code

O @ e um mecanismo de referencia direta a arquivos disponivel tanto no terminal quanto na extensao VS Code. Ao digitar @ seguido de um caminho de arquivo, o Claude Code localiza esse arquivo no seu projeto, le seu conteudo e o inclui como contexto na conversa. Isso e fundamentalmente diferente de copiar e colar trechos de codigo no prompt — o agente recebe o arquivo completo e atualizado, com todas as suas dependencias implicitas.

Na documentacao oficial, o @ aparece como um dos quick commands do modo interativo: ao digitar @ no prompt, o Claude Code ativa o autocomplete de caminhos de arquivo, com suporte a fuzzy matching. Voce nao precisa digitar o caminho completo — basta digitar parte do nome e o Claude encontra o arquivo correto.

Fuzzy matching: O Claude Code suporta correspondencia aproximada nos @-mentions. Digitar @auth pode corresponder a auth.js, AuthService.ts, middleware/auth.guard.ts, etc. Voce nao precisa lembrar o caminho exato.

Como o @ resolve para conteudo real

O @ transforma um caminho de arquivo em conteudo real injetado no contexto, permitindo ao Claude replicar padroes com fidelidade.

@ no Terminal vs. na Extensao VS Code

O operador @ funciona em ambos os ambientes, mas com capacidades ligeiramente diferentes. No terminal, o @ ativa autocomplete de caminhos de arquivo com Tab. Na extensao VS Code, voce ganha recursos adicionais como selecao de ranges de linha e integracao com o editor visual.

Funcionalidade Terminal (CLI) Extensao VS Code
@-mention de arquivo Sim — digita @ e o caminho Sim — com fuzzy matching visual
Autocomplete com Tab Sim — Tab completa o caminho Nao (usa fuzzy match no lugar)
Range de linhas Manual: @file.ts#5-10 Automatico: selecione no editor + Option+K / Alt+K
Referencia a pastas Sim — @src/components/ Sim — com trailing slash
Selecao do editor N/A Claude ve automaticamente o texto selecionado no editor
Referencia a terminal N/A @terminal:nome — inclui output do terminal
Drag & drop N/A Segure Shift e arraste arquivos para o prompt
Atalho essencial no VS Code: Selecione um trecho de codigo no editor e pressione Option+K (Mac) ou Alt+K (Windows/Linux). Isso insere automaticamente uma referencia como @app.ts#5-10 no prompt, apontando exatamente para as linhas selecionadas.

Tab autocomplete no terminal

No modo interativo do CLI, ao digitar @ seguido de caracteres, o Claude Code oferece autocomplete de caminhos de arquivo via Tab. Isso funciona com fuzzy matching — voce nao precisa digitar o caminho completo.

Descrever um padrao vs. Referenciar um padrao

Uma das mudancas mais impactantes na forma de usar o Claude Code e trocar a descricao textual por referencia direta. Quando voce descreve um padrao com palavras, o Claude precisa interpretar sua descricao e pode errar detalhes. Quando voce aponta para o arquivo real, ele le exatamente como o padrao esta implementado.

X Descrever o padrao

Claude interpreta sua descricao — pode haver ambiguidade
Voce precisa lembrar e explicar cada detalhe do padrao
Detalhes como imports, tipos e error handling ficam de fora
Prompts longos e frageis — facil esquecer algo
Resultado generico, nao alinhado com o projeto
vs

@ Referenciar o padrao

Claude le o arquivo real — zero ambiguidade
Voce so aponta para o exemplo — o Claude extrai o padrao
Todos os detalhes sao capturados: imports, tipos, estilo
Prompts curtos e precisos — menos chance de erro
Resultado consistente com as convencoes existentes

15 palavras vs 120 palavras: O prompt com @ e mais curto, mais preciso e produz resultado mais consistente. O Claude le o UserController real, extrai todos os padroes (imports, tipos, error handling, formatacao de resposta) e os replica no novo arquivo.

Casos de uso com @-mentions

1 Novo componente

Criar componente React seguindo a estrutura de um componente existente que ja usa seus patterns de styling, hooks e types.

2 Testes consistentes

Gerar testes para um modulo novo usando o mesmo padrao (setup, mocks, assertions) de um arquivo de testes existente.

3 Replicar estrutura

Criar um novo modulo (service, repository, routes) replicando a mesma arquitetura de um modulo ja implementado.

4 Refatorar seguindo exemplo

Refatorar um arquivo legado para seguir o padrao de um arquivo ja modernizado no mesmo projeto.

5 Migrar tecnologia

Converter um componente class-based para hooks seguindo um exemplo de componente que ja foi migrado.

6 Documentar padrao

Gerar JSDoc ou comentarios seguindo o estilo de documentacao de um arquivo referenciado.

Combinando @ com instrucoes claras

O poder real do @ aparece quando voce combina referencias a arquivos com instrucoes claras e especificas. O padrao mais eficaz e: referencia + objetivo + restricoes.

Padrao de prompt Exemplo
Criar seguindo padrao Crie X seguindo o padrao de @arquivo
Refatorar com referencia @arquivo-legado — refatore usando o padrao de @arquivo-moderno
Comparar e alinhar Compare @arquivo-A e @arquivo-B e alinhe A com o padrao de B
Extender padrao existente Adicione metodo X em @arquivo seguindo o padrao dos metodos existentes
Aplicar padrao a multiplos Aplique o padrao de error handling de @middleware em @controller-A e @controller-B
Dica de produtividade: Quando voce tem um modulo bem implementado que serve de "gold standard", referencie-o em todos os prompts de criacao de modulos similares. Isso garante consistencia automatica em todo o codebase, sem precisar documentar cada convencao manualmente.

Por que referenciar e melhor que descrever

A documentacao oficial de best practices da Anthropic recomenda fornecer instrucoes especificas que apontem para exemplos existentes. A razao e simples: o Claude Code tem acesso ao filesystem e pode ler seus arquivos diretamente. Aproveitar essa capacidade e fundamental para obter resultados consistentes com o padrao do seu projeto.

1 Fidelidade total ao padrao

O Claude le o arquivo real com todos os detalhes — nao depende de descricao incompleta. Imports, tipos, naming conventions, espacamento, tudo e capturado.

2 Prompts menores e melhores

Em vez de 100+ palavras descrevendo um padrao, voce usa 15 palavras com um @-mention. Menos texto no prompt = mais espaco na context window para o conteudo real.

3 Sempre atualizado

Se o arquivo de referencia mudar, o proximo prompt com @ le a versao mais recente. Descricoes textuais ficam desatualizadas e causam drift.

4 Consistencia no time

Todos os devs do time podem usar o mesmo @-mention para referenciar o mesmo padrao. Nao depende de cada pessoa descrever o padrao de forma diferente.

Armadilha comum: Copiar e colar trechos de codigo no prompt em vez de usar @. Quando voce cola codigo, ele pode estar desatualizado, incompleto ou fora de contexto. O @ sempre le a versao atual do arquivo, com todo o contexto ao redor.

Combinando @ com CLAUDE.md

O @ funciona perfeitamente em conjunto com o CLAUDE.md do seu projeto. Enquanto o CLAUDE.md documenta regras gerais e convencoes (como "use Zod para validacao" ou "siga o padrao Repository"), o @ aponta para a implementacao concreta dessas regras. Os dois mecanismos se complementam:

CLAUDE.md do projeto Regras gerais 
# Convencoes do projeto
- Controllers seguem o padrao em src/controllers/UserController.ts
- Services usam injecao de dependencia via construtor
- Testes seguem o padrao de tests/services/UserService.test.ts
- Schemas Zod ficam em src/schemas/ (ver email.schema.ts como exemplo)

Com essas regras no CLAUDE.md, o Claude Code ja sabe quais arquivos referenciar. Mas voce pode reforcar ou sobrepor essa orientacao no prompt com @ quando quiser apontar para um arquivo especifico diferente.

Dica: Use o comando /init para gerar um CLAUDE.md inicial e depois adicione referencias a arquivos-padrao manualmente. Isso cria um "guia de estilo vivo" que o Claude Code consulta automaticamente em toda conversa.
