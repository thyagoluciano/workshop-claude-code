Modulo 4 . Topico 3

Passar URLs, imagens, logs via pipe

O pipe (|) transforma o Claude Code em mais uma peca do seu pipeline Unix. Alimente-o com logs, diffs, outputs de testes, paginas web ou imagens — e receba analises contextualizadas sem sair do terminal.

O poder do pipe: contexto externo direto no prompt

O modo nao-interativo (-p) do Claude Code le stdin automaticamente. Isso significa que qualquer comando que produza texto pode alimentar o Claude diretamente, usando o operador | do shell — exatamente como voce faria com grep, awk ou jq. O Claude se torna mais uma ferramenta composivel no seu toolkit Unix.

A vantagem sobre colar conteudo manualmente no chat e tripla: automacao (scripts e CI podem usar), volume (ate 10 MB de stdin) e composicao (voce encadeia com outros comandos antes e depois do Claude).

Qualquer comando que produza texto pode alimentar o Claude via pipe. O output pode ir para o terminal, um arquivo ou ser parseado como JSON.

Sintaxe basica

O padrao fundamental e simples: comando | claude -p "sua pergunta". A flag -p (ou --print) ativa o modo nao-interativo: o Claude processa o input, imprime a resposta e encerra. Sem sessao, sem interface interativa.

Padrao basico do pipe bash 
# Padrao geral
comando | claude -p "sua pergunta sobre o conteudo"

# Com redirecionamento de saida
comando | claude -p "sua pergunta" > resultado.txt

# Com output JSON estruturado
comando | claude -p "sua pergunta" --output-format json

# Com --bare para scripts e CI (pula discovery de plugins, hooks, MCP)
comando | claude --bare -p "sua pergunta" --allowedTools "Read"
--bare para scripts: Em CI e automacoes, use --bare junto com -p. Isso pula o auto-discovery de hooks, skills, plugins, MCP servers e CLAUDE.md, garantindo que o resultado seja o mesmo em qualquer maquina. E o modo recomendado pela Anthropic para chamadas programaticas.

Caso de uso: analise de logs

Um dos usos mais poderosos do pipe e alimentar logs de erro diretamente ao Claude. Em vez de ler manualmente centenas de linhas de stack trace, deixe o modelo identificar a causa raiz e sugerir correcoes.

~/projeto
$ cat error.log | claude -p "O que causou esse erro? Sugira uma correção."

Analisando o log, o erro ocorre na linha 142 de auth.service.ts...
A causa raiz e um token JWT expirado que nao esta sendo renovado...
Sugestao: adicione um middleware de refresh token antes da validacao...

$ tail -100 /var/log/nginx/error.log | claude -p "Resuma os erros mais frequentes"

$ cat build-error.txt | claude -p "Explique a causa raiz deste erro de build" > diagnostico.txt

Caso de uso: revisao de diffs

Pipe o diff das suas mudancas para o Claude revisar antes de abrir um PR. Ele pode encontrar bugs, sugerir melhorias e ate servir como linter personalizado.

Revisao de codigo com diffs bash 
# Revisar mudancas staged
git diff --staged | claude -p "Revise essas mudancas. Aponte bugs, melhorias e problemas de seguranca."

# Comparar com a branch main
git diff main | claude -p "Resuma o que mudou e se ha breaking changes."

# Usar como linter de typos em package.json
git diff main | claude -p "Voce e um linter de typos. Para cada typo neste diff, reporte arquivo:linha em uma linha e o problema na seguinte. Nada mais."

# Diff de um PR especifico via GitHub CLI
gh pr diff 42 | claude -p "Revise este PR focando em seguranca."
Dica de CI: A documentacao oficial mostra como adicionar revisao por Claude como script do package.json: "lint:claude": "git diff main | claude -p \"voce e um linter de typos...\"". O diff e passado via pipe, entao o Claude nao precisa de permissao Bash para le-lo.

Caso de uso: output de testes

Quando seus testes falham, o output pode ser extenso e confuso. Pipe o resultado diretamente para o Claude explicar quais testes falharam e por que.

Diagnostico de falhas de teste bash 
# Capturar stdout + stderr de testes
npm test 2>&1 | claude -p "Explique quais testes falharam e sugira correcoes."

# Testes Python com pytest
pytest --tb=short 2>&1 | claude -p "Analise as falhas e priorize por severidade."

# Go tests
go test ./... 2>&1 | claude -p "Quais pacotes falharam e qual a causa provavel?"

# Capturar tipo check do TypeScript
npx tsc --noEmit 2>&1 | claude -p "Explique esses erros de tipo e como corrigi-los."

Caso de uso: URLs e conteudo web

Voce pode buscar conteudo de URLs com curl e passar via pipe para o Claude analisar. Isso e util para resumir documentacao, analisar respostas de API ou processar paginas web.

Trabalhando com URLs bash 
# Buscar e resumir uma pagina web
curl -s https://example.com/docs/api | claude -p "Resuma os endpoints principais desta API."

# Analisar resposta de uma API JSON
curl -s https://api.exemplo.com/health | claude -p "Este health check indica algum problema?"

# Passar URL diretamente no prompt interativo (sem pipe)
claude
# > Analise a documentacao em https://docs.exemplo.com/auth

# Referenciar com @ no modo interativo
# > Explique a logica em @src/utils/auth.js
Referencia com @: No modo interativo, voce pode usar @ para referenciar arquivos locais (@src/auth.js), diretorios (@src/components) e ate recursos MCP (@github:repos/owner/repo/issues). O conteudo e incluido automaticamente no contexto da conversa.

Imagens e screenshots: Claude Code e multimodal

O Claude Code aceita imagens diretamente — ele e multimodal. Voce pode arrastar imagens para a janela, colar do clipboard ou fornecer o caminho do arquivo. Isso e especialmente util para analisar screenshots de erros, mockups de UI e diagramas de arquitetura.

1 Arrastar e soltar

Arraste uma imagem diretamente para a janela do Claude Code no terminal.

2 Colar do clipboard

No macOS: Cmd+Ctrl+Shift+4 para capturar, depois Ctrl+V para colar no Claude.

3 Caminho do arquivo

Forneca o path: Analise esta imagem: /path/to/screenshot.png

4 Verificacao visual

Cole screenshots para o Claude comparar output visual com o esperado — ideal para frontend.

Exemplos com imagens claude (interativo) 
# No modo interativo, apos colar uma imagem:
"O que esta errado neste screenshot de erro?"

# Gerar CSS a partir de um mockup:
"Gere o CSS para replicar este design mockup"

# Analisar diagrama de arquitetura:
"Analise este diagrama. Quais gargalos voce identifica?"

# Fornecer path da imagem:
"Compare este screenshot @screenshot-antes.png com @screenshot-depois.png"
Dica de frontend: A documentacao oficial destaca que o Claude performa dramaticamente melhor quando pode verificar seu proprio trabalho visualmente. Para desenvolvimento de UI, cole screenshots do resultado para o Claude comparar com o esperado e iterar.

Combinando multiplas fontes

Voce pode combinar multiplas fontes de dados usando tecnicas do shell como process substitution, here strings e concatenacao para dar mais contexto ao Claude.

Tecnicas avancadas de composicao bash 
# Concatenar multiplos arquivos
cat src/auth.ts src/auth.test.ts | claude -p "Os testes cobrem todos os cenarios do codigo?"

# Pipar log + pedir para Claude buscar mais contexto com suas ferramentas
cat deploy.log | claude -p "Analise este log de deploy. Use suas ferramentas para ler arquivos relevantes do projeto e diagnosticar a causa raiz."

# Encadear com jq para filtrar antes de passar ao Claude
curl -s https://api.exemplo.com/users | jq '.[] | select(.active == false)' | claude -p "Por que esses usuarios estao inativos?"

# Process substitution para comparar dois outputs
diff <(git show main:src/config.ts) <(cat src/config.ts) | claude -p "O que mudou na configuracao?"

# Continuacao de conversa com pipe (manter contexto)
claude -p "Revise este codebase"
cat new-errors.log | claude -p "Novos erros apareceram. Analise." --continue

One-liners poderosos

Aqui estao exemplos de one-liners que combinam pipe com o Claude para tarefas do dia a dia de um desenvolvedor.

one-liners
# Gerar commit message automaticamente
$ git diff --staged | claude -p "Gere uma commit message concisa seguindo Conventional Commits"

# Resumir commits recentes para standup
$ git log --oneline -20 | claude -p "Resuma esses commits para um relatorio de standup"

# Auditar Dockerfile
$ cat Dockerfile | claude -p "Revise este Dockerfile para seguranca e boas praticas"

# Explicar um script complexo
$ cat deploy.sh | claude -p "Explique este script passo a passo"

# Gerar documentacao de API a partir do schema
$ cat openapi.yaml | claude -p "Gere docs em Markdown para cada endpoint" > api-docs.md

# Extrair funcoes e salvar como JSON estruturado
$ cat src/utils.ts | claude -p "Extraia os nomes das funcoes" --output-format json | jq -r '.result'

Limitacoes e cuidados

O pipe e poderoso, mas tem limites que voce precisa conhecer para evitar surpresas.

! Limite de 10 MB no stdin

A partir da versao v2.1.128, o stdin via pipe e limitado a 10 MB. Se exceder, o Claude encerra com erro e status nao-zero. Para inputs maiores, salve em arquivo e referencie o path no prompt.

! Encoding e binarios

O pipe funciona com texto (UTF-8). Arquivos binarios, imagens via pipe ou conteudo com encoding exotico podem nao funcionar. Para imagens, use o modo interativo (arrastar, colar, ou fornecer path).

! Sem sessao persistente

O -p roda sem sessao por padrao. Use --continue para encadear com a conversa anterior, ou capture o session_id via --output-format json para retomar depois com --resume.

! Context window

Mesmo dentro do limite de 10 MB, o conteudo do stdin consome tokens da context window. Filtre o input antes de passar ao Claude (use grep, tail, jq) para manter o foco.

Alternativa para arquivos grandes: Se o input exceder 10 MB, salve-o em disco e referencie no prompt: claude -p "Analise o arquivo em /tmp/big-log.txt". O Claude usara a ferramenta Read para ler o arquivo sob demanda, consumindo menos contexto do que pipar tudo via stdin.

Quando usar pipe vs. modo interativo

Cenario Pipe (-p) Interativo
CI/CD e automacao Ideal — sem interacao humana Nao aplicavel
Pre-commit hooks Ideal — rapido e automatico Muito lento
Analise rapida de log Uma linha, resultado direto Precisa colar conteudo
Tarefa complexa multi-step Limitado a 1 turno Ideal — loop agentico
Imagens e screenshots Nao suportado via pipe Arrastar, colar ou @path
Exploracao de codebase Sem ferramentas de busca Grep, Glob, Read disponiveis
Scripts em batch Composivel com shell Manual
