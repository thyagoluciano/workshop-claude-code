Loop Agêntico: como o Claude explora, planeja e implementa

Entenda o ciclo fundamental que diferencia o Claude Code de um simples chatbot: um agente que lê código, toma decisões, usa ferramentas, avalia resultados e se autocorrige — em loop.

O que é o Loop Agêntico?

O Claude Code não funciona como um autocomplete inteligente. Ele é um agente autônomo que opera em ciclo contínuo: recebe um objetivo do desenvolvedor, usa ferramentas reais (bash, grep, leitura/escrita de arquivos, git), avalia o resultado de cada ação e decide o próximo passo — tudo sem intervenção humana entre os passos.

Na definição da própria Anthropic, um agente é um LLM usando ferramentas de forma autônoma em um loop. O desenvolvedor define o objetivo e mantém controle sobre o que é commitado, mas o ciclo de execução roda de forma independente.

Definição oficial: "An agentic system acts toward a goal with a degree of autonomy, rather than responding to one prompt at a time. Claude Code reads a codebase, plans a sequence of actions, executes them using real development tools, evaluates the result, and adjusts its approach."
— Anthropic, Claude Code Best Practices

Anatomia do Loop

Cada interação com o Claude Code dispara um ciclo que se repete até que o objetivo seja atingido (ou o limite de iterações seja alcançado). Veja as fases:

O loop continua até o objetivo ser alcançado ou o limite de iterações ser atingido

As 4 Fases em Detalhe

🔍 Fase 1 — Explorar (Context Gathering)

Antes de escrever uma única linha de código, o Claude lê e navega o codebase. Ele usa ferramentas como grep, glob e Read para entender a estrutura do projeto, encontrar arquivos relevantes e mapear dependências.

A Anthropic descreve essa abordagem como "just in time" context: em vez de carregar todo o código de uma vez, o agente mantém referências leves (caminhos de arquivo, queries, links) e carrega dados sob demanda. Isso espelha a cognição humana — não memorizamos corpora inteiros, mas usamos sistemas de indexação para recuperar informação quando necessário.

Exemplo: o que o Claude faz internamente ao receber "fix the auth bug" Ferramentas usadas 
// 1. Busca por arquivos relacionados a autenticação
glob("**/auth*.{ts,js,py}")
grep("authentication", type="ts")

// 2. Lê os arquivos encontrados
Read("src/middleware/auth.ts")
Read("src/services/authService.ts")
Read("tests/auth.test.ts")

// 3. Examina logs ou erros mencionados
bash("cat logs/error.log | tail -50")

// 4. Verifica testes existentes
bash("npm test -- --grep auth 2>&1 | tail -20")
Conceito-chave: Progressive Disclosure. Cada interação com uma ferramenta revela contexto que informa a próxima decisão. Tamanhos de arquivo sugerem complexidade; convenções de nomes indicam propósito; timestamps servem como proxy para relevância. O agente constrói entendimento camada por camada.

📋 Fase 2 — Planejar (Strategy Formation)

Com o contexto coletado, o Claude usa extended thinking (cadeia de pensamento interna) para formular uma estratégia antes de agir. Isso inclui decidir quais arquivos modificar, em que ordem, e quais verificações rodar para validar as mudanças.

Essa fase é crucial: sem planejamento, o agente tende a "one-shot" — tentar resolver tudo de uma vez, gerando código frágil e difícil de debugar. A Anthropic descobriu isso ao construir agentes de longa duração e implementou estratégias para forçar o progresso incremental.

Pattern

Progresso Incremental vs. One-shot

AbordagemComportamentoResultado
❌ One-shot Tenta implementar tudo de uma vez Estoura o contexto, deixa código quebrado
✓ Incremental Uma feature por vez, commit, próxima Progresso estável, estado limpo entre passos

⚡ Fase 3 — Executar (Tool Use)

O Claude age sobre o codebase usando as mesmas ferramentas que um programador usa diariamente: edição de arquivos, execução de comandos shell, interação com Git. A Anthropic descreve esse princípio como: "Claude needs the same tools that programmers use every day."

📝 Edição de Código

Edit e Write — modifica arquivos existentes ou cria novos, aplicando diffs precisos.

💻 Shell / Bash

Roda comandos arbitrários: instalar deps, rodar builds, executar scripts, verificar outputs.

🔀 Git

Commits, branches, diffs, revert. O agente pode criar PRs e escrever mensagens descritivas de commit.

🔌 MCP Servers

Ferramentas externas via Model Context Protocol: bancos de dados, APIs, Jira, Slack, etc.

Exemplo: Claude corrigindo um bug e validando Execução real 
// Após explorar e planejar, o Claude executa:

// 1. Aplica o fix
Edit("src/middleware/auth.ts", {
  old: "if (token.expired)",
  new: "if (token.expired || !token.valid)"
})

// 2. Roda os testes
bash("npm test -- --grep 'auth' 2>&1")

// 3. Se os testes passam, commita
bash("git add -A && git commit -m 'fix: validate token before auth check'")

✓ Fase 4 — Avaliar (Verify & Iterate)

Após cada ação, o Claude avalia o resultado. Se um teste falha, se o build quebra, ou se o output não corresponde ao esperado, ele volta ao início do loop — relê o erro, replaneja e tenta de novo. Esse ciclo de feedback é o que torna o sistema genuinamente agêntico.

O loop continua até uma de duas condições: o Claude responde sem solicitar nenhuma ferramenta (indicando que a tarefa foi concluída) ou o limite máximo de iterações é atingido.

Lição prática: A avaliação é tão importante quanto a execução. Sempre inclua critérios de verificação no seu prompt: "rode os testes", "faça o build", "mostre o output". Isso dá ao Claude um sinal claro de sucesso ou falha para guiar o loop.

Ferramentas que compõem o Loop

O Claude Code vem equipado com um conjunto de ferramentas nativas que ele usa durante o loop. Entender quais estão disponíveis ajuda a escrever prompts melhores:

FerramentaFaseO que faz
Glob Explorar Busca arquivos por padrão (ex: **/*.test.ts)
Grep Explorar Busca conteúdo em arquivos com regex
Read Explorar Lê arquivos específicos (com offset/limit para arquivos grandes)
Bash Executar Executa comandos shell arbitrários

O Loop e o Context Engineering

O loop agêntico gera dados a cada iteração: resultados de ferramentas, conteúdo de arquivos, outputs de comandos. Tudo isso consome tokens da context window — o recurso mais precioso e limitado do agente.

A Anthropic cunhou o termo context engineering para descrever a arte de curar o menor conjunto possível de tokens de alto sinal que maximize a probabilidade de um resultado desejado. Enquanto prompt engineering foca em como escrever instruções, context engineering foca no estado holístico disponível ao modelo a cada turno.

Estratégias para Loops de Longa Duração

Quando uma tarefa é complexa demais para caber em uma única context window, o Claude Code usa três estratégias complementares, todas documentadas pela Anthropic:

📦 Compaction

Resumir o histórico preservando decisões arquiteturais, bugs abertos e detalhes de implementação. Descarta tool outputs redundantes.

📝 Note-taking

O agente escreve notas estruturadas (ex: todo-list, progress file) persistidas fora da context window e recuperadas depois.

🤖 Sub-agentes

Tarefas focadas delegadas a agentes especializados, cada um com context window limpa. Retornam resumos condensados ao agente principal.

Harness para agentes de longa duração: A Anthropic desenvolveu um padrão com dois componentes: um initializer agent que configura o ambiente na primeira execução (feature list, init.sh, progress file), e um coding agent que faz progresso incremental em cada sessão, deixando artefatos claros para a próxima sessão.

Modelo Mental: Engenheiro em Turnos

A analogia oficial da Anthropic é poderosa: imagine um projeto staffado por engenheiros trabalhando em turnos, onde cada novo engenheiro chega sem memória do turno anterior. A context window é o "turno" — limitada e sem carryover automático.

O que um bom engenheiro faz ao chegar num turno? Lê as notas do turno anterior. Verifica o git log. Roda os testes para ver o estado atual. E antes de sair, documenta o que fez. É exatamente assim que o Claude Code opera quando bem configurado.

Sessão típica de um agente de longa duração Padrão documentado pela Anthropic 
// Início do turno — orientação
bash("pwd")                                    // Onde estou?
Read("claude-progress.txt")                  // O que foi feito?
bash("git log --oneline -20")                // Histórico recente
Read("feature_list.json")                    // O que falta fazer?

// Verificação de saúde
bash("./init.sh")                             // Sobe o servidor
// [testa funcionalidade básica]

// Trabalho — uma feature por vez
// [implementa, testa, commita]

// Fim do turno — deixa estado limpo
bash("git add -A && git commit -m '...'")
Edit("claude-progress.txt", ...)           // Atualiza notas

Resumo

1.

O Claude Code é um agente, não um chatbot. Ele opera em loop autônomo: explora → planeja → executa → avalia → repete.

2.

Ferramentas reais, não simulação. O agente usa grep, bash, git, file I/O — as mesmas ferramentas que você usa.

3.

Context é recurso finito. Cada token importa. Context engineering supera prompt engineering em ambientes agênticos.

4.

Para loops longos: compaction + notas + subagentes. A analogia do "engenheiro em turnos" guia o design de sessões eficazes.

5.

Sempre inclua critérios de verificação. Sem sinais claros de sucesso/falha, o agente não sabe quando parar ou corrigir.
