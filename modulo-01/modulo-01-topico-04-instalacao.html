Instalação, Configuração e Primeiros Passos

Do npm install ao primeiro prompt produtivo — tudo que você precisa para configurar o Claude Code no terminal, na IDE e entender os comandos essenciais.

Pré-requisitos

Antes de instalar, verifique se seu ambiente atende aos requisitos mínimos. O Claude Code roda localmente no seu terminal e se conecta diretamente às APIs de modelo — não precisa de servidor backend, índice remoto de código ou configuração complexa.

✅ Sistema Operacional

macOS, Linux ou Windows (via WSL). Suporte nativo a Bash, Zsh, PowerShell e CMD.

✅ Node.js 18+

Necessário apenas para instalação via npm. O instalador nativo inclui seu próprio runtime.

✅ Conta Claude

Pro, Max, Team, Enterprise ou Console (API). O plano gratuito não inclui Claude Code.

Instalação

Existem três formas de instalar o Claude Code. O instalador nativo é o método recomendado pela Anthropic — ele baixa o binário diretamente sem precisar de Node.js. O pacote npm instala o mesmo binário nativo, apenas usando npm como mecanismo de distribuição.

Opção 1: Instalador nativo (recomendado)

macOS / Linux Terminal 
# Instala a versão estável mais recente
curl -fsSL https://claude.ai/install.sh | bash
Windows (PowerShell) Terminal 
# Instala via PowerShell
irm https://claude.ai/install.ps1 | iex

Opção 2: NPM (requer Node.js 18+)

npm Terminal 
# Instalar globalmente
npm install -g @anthropic-ai/claude-code

# Atualizar para a última versão
npm install -g @anthropic-ai/claude-code@latest

Opção 3: Homebrew (macOS)

Homebrew Terminal 
brew install --cask claude-code
Nunca use sudo na instalação! Executar sudo npm install -g pode causar problemas de permissão e riscos de segurança. Se encontrar erros de permissão no npm, configure o diretório global do npm para um local no seu usuário.

Autenticação

Após a instalação, execute claude no terminal. Na primeira execução o Claude Code abre o navegador para autenticação via sua conta Claude. Existem dois caminhos de autenticação:

👤 Plano de assinatura (Pro/Max/Team/Enterprise)

Login via browser com sua conta claude.ai. Usa o mesmo plano do chat web/desktop/mobile — uma assinatura unificada. Ao fazer login, um workspace "Claude Code" é criado automaticamente no Console.

🔑 API Key (Console / Pay-as-you-go)

Se a variável ANTHROPIC_API_KEY estiver definida no ambiente, o Claude Code usará a API key em vez do login por assinatura. Neste caso, a cobrança será por token consumido via API.

Atenção com a ANTHROPIC_API_KEY: Se você tem um plano Pro/Max e também uma API key configurada como variável de ambiente, o Claude Code vai priorizar a API key — gerando cobranças à parte. Use /login dentro do Claude Code para alternar para seu plano de assinatura.

Modos de uso do CLI

O Claude Code oferece várias formas de invocação além do REPL interativo. Cada uma é otimizada para um cenário diferente:

Comando Modo Cenário
claude REPL interativo Sessão de trabalho contínua — o modo mais comum
claude "query" REPL com prompt inicial Iniciar sessão já com uma tarefa definida
claude -p "query" One-shot (SDK/pipe) Executa a query, retorna resultado e sai — ideal para scripts
cat file | claude -p "..." Pipe + one-shot Alimenta conteúdo via stdin — logs, erros, CSVs
claude -c Continue Retoma a conversa mais recente
claude -c -p "query" Continue + one-shot Continua a conversa anterior via SDK e sai

Exemplos práticos de pipe

Cenários comuns com pipe Terminal 
# Analisar log de erro
cat error.log | claude -p "O que causou esse erro? Sugira a correção."

# Revisar diff antes de commitar
git diff --staged | claude -p "Revise esse diff. Algum problema?"

# Explicar output de teste
npm test 2>&1 | claude -p "Explique os testes que falharam e como corrigir."

# Gerar resumo de um CSV
cat vendas.csv | claude -p "Resuma os dados. Top 5 produtos por receita."

Comandos essenciais (/slash commands)

Dentro de uma sessão interativa, os slash commands controlam o comportamento do Claude Code. Eles não são enviados ao modelo como prompt — são comandos internos do sistema.

Comando O que faz Quando usar
/help Exibe lista de comandos e atalhos disponíveis Sempre que esquecer um comando
/init Gera um CLAUDE.md inicial para o projeto Primeira vez configurando Claude Code num projeto
/model Alterna entre modelos (Opus, Sonnet, Haiku) Trocar para um modelo mais rápido/barato ou mais capaz
/effort Ajusta o nível de esforço do modelo (thinking) Tarefas simples (reduzir) ou complexas (aumentar)
/clear Limpa toda a conversa e começa do zero Ao trocar de tarefa — evita context rot
/compact Compacta o histórico preservando o essencial Quando o contexto está ficando grande mas quer manter continuidade
/config Abre o menu de configurações do Claude Code Ajustar thinking, permissões, status line etc.
/hooks Navega e gerencia hooks configurados Verificar/editar automações de quality gate
/login Alterna o método de autenticação Mudar de API key para plano de assinatura (ou vice-versa)
/usage Mostra o consumo de tokens da sessão Monitorar custos durante uma sessão longa

Flags do CLI mais usadas

Além do uso interativo, o CLI aceita flags que controlam comportamento, permissões e ferramentas disponíveis. Aqui estão as mais importantes para o dia a dia:

Flags essenciais CLI Reference 
# Iniciar com prompt e sair (modo SDK/pipe)
claude -p "Explique a função calculateTax"

# Continuar a conversa mais recente
claude -c

# Adicionar diretórios extras ao contexto
claude --add-dir ../shared-libs --add-dir ../config

# Pré-aprovar ferramentas específicas (sem prompt de confirmação)
claude --allowedTools "Bash(git log:*)" "Bash(git diff:*)" "Read"

# Bloquear ferramentas específicas
claude --disallowedTools "Bash(rm:*)" "Write"

# Definir o modo de permissão
claude --permission-mode plan       # Somente leitura
claude --permission-mode default    # Pede confirmação (padrão)
claude --permission-mode auto       # Classificador decide
Dica para times: Use --allowedTools e --disallowedTools em scripts de CI ou wrappers do time para garantir que o Claude Code opere apenas com ferramentas aprovadas. Combinado com --permission-mode dontAsk, ferramentas não listadas são negadas automaticamente.

Custos: o que esperar

O Claude Code consome tokens por sessão — cada leitura de arquivo, cada resposta, cada pensamento estendido conta. Os custos variam com o modelo escolhido e a intensidade de uso.

~$13

Custo médio por desenvolvedor por dia ativo
(média enterprise, Anthropic 2025)

$150-250

Faixa típica por desenvolvedor por mês
(uso regular em projetos reais)

<$30

90% dos usuários ficam abaixo desse valor por dia
(percentil 90, Anthropic 2025)

Custo por modelo (API - pay-as-you-go)

Modelo Input (por 1M tokens) Output (por 1M tokens) Perfil de uso
Haiku 4.5 $0.80 $4 Tarefas simples, alto volume, baixo custo
Sonnet 4.6 $3 $15 Melhor custo-benefício para coding diário
Opus 4.6 / 4.7 $5 $25 Arquitetura complexa, reasoning profundo

Prompt caching pode reduzir custos de input em até 90%. Batch processing reduz em até 50%. Use /usage para monitorar consumo em tempo real.

Dica de economia: Use /clear ao trocar de tarefa (evita arrastar contexto velho), /compact em sessões longas (comprime mantendo essencial), e /model para alternar entre Sonnet (dia a dia) e Opus (decisões críticas). Reduza o effort level com /effort para tarefas triviais.

Checklist: pronto para usar

Use esta lista para validar que tudo está configurado antes de começar a trabalhar:

☑ Claude Code instalado (claude --version retorna versão)
☑ Autenticação feita (login via browser ou API key configurada)
☑ Terminal aberto no diretório do projeto (cd ~/meu-projeto)
☑ Projeto com git inicializado (git status funciona)
☑ CLAUDE.md criado na raiz (ou usar /init para gerar)
☑ Modelo selecionado (/model — Sonnet 4.6 é o padrão recomendado)
☑ Extensão da IDE instalada (opcional — VS Code ou JetBrains)

Atalhos de teclado no REPL

Dentro da sessão interativa, estes atalhos aceleram o fluxo de trabalho:

⌨ Navegação

Escape — cancela a operação em andamento
Ctrl+C — interrompe a resposta do modelo
↑ / ↓ — navega pelo histórico de prompts

⚒ Contexto

@ — referencia arquivos ou símbolos do projeto
Tab — autocomplete para caminhos de arquivo
Shift+Enter — quebra de linha (multi-line input)
