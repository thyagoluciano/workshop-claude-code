Diferença entre Chatbot e Agente de Código

Um chatbot responde perguntas. Um agente de código resolve problemas — ele explora, planeja, executa, verifica e itera. Entender essa diferença muda a forma como você interage com o Claude Code.

Definições: o que a Anthropic chama de "agente"?

A Anthropic define formalmente a distinção no paper "Building Effective Agents" (dez/2024): workflows são sistemas onde LLMs e ferramentas são orquestrados por caminhos de código predefinidos; agents são sistemas onde o LLM dirige dinamicamente seus próprios processos e uso de ferramentas, mantendo controle sobre como realiza as tarefas.

Um chatbot tradicional é o caso mais simples de workflow: recebe uma mensagem, gera uma resposta, e para. Não tem estado entre turnos, não executa ações no mundo real e não verifica seus resultados. Um agente como o Claude Code opera em um loop contínuo onde ele decide quais ferramentas usar, observa os resultados e ajusta o plano — tudo de forma autônoma.

Definição oficial: "An agent is an AI model that directs its own processes and tool use when accomplishing a task — deciding for itself how to achieve what users want, rather than following a fixed script."
— Anthropic, "Measuring AI Agent Autonomy in Practice" (2025)

Chatbot vs. Agente: comparação lado a lado

💬 Chatbot (LLM puro)

Responde perguntas — o usuário dirige a conversa
Sem acesso a ferramentas externas (file system, bash, git)
Cada turno é independente — sem "memória de trabalho" entre ações
Gera texto (código incluído), mas não o executa nem o valida
Não pode verificar se a resposta está correta
Fluxo linear: pergunta → resposta → fim
vs

🤖 Agente de Código (Claude Code)

Resolve tarefas — o agente dirige a execução
Acesso a ferramentas reais: Bash, Edit, Read, Grep, Glob
Mantém estado — cada ação informa a próxima dentro do mesmo loop
Escreve código, executa, lê o output e corrige erros
Pode rodar testes, linters e builds para auto-verificar
Loop agêntico: observar → planejar → agir → verificar → repetir

Os dois fluxos na prática

A diferença fundamental não é a qualidade do modelo — é o ciclo de feedback. Um chatbot gera uma resposta e encerra. Um agente gera, executa, observa o resultado e decide o próximo passo. Esse loop é o que permite ao Claude Code resolver problemas complexos que exigem múltiplas etapas.

O chatbot termina após gerar texto. O agente itera até que o resultado atenda ao critério de verificação.

O toolbelt do Claude Code

O que torna o Claude Code um agente (e não um chatbot sofisticado) são as ferramentas reais que ele pode invocar. O princípio de design por trás do Claude Code é que o modelo precisa das mesmas ferramentas que um programador usa no dia a dia: encontrar arquivos, ler código, editar, executar, debugar — e às vezes fazer tudo isso iterativamente até o código funcionar.

Ferramenta O que faz Equivalente humano
Read Lê o conteúdo de um arquivo específico Abrir arquivo no editor
Glob Busca arquivos por padrão de nome (**/*.tsx) find . -name "*.tsx"
Grep Busca texto/regex dentro dos arquivos rg "pattern" --type ts
Edit Faz substituições cirúrgicas em arquivos Find & Replace preciso
Write Cria ou sobrescreve um arquivo completo Criar novo arquivo
Bash Executa qualquer comando no terminal Terminal / shell do dev
MCP Tools Conecta com APIs externas (GitHub, Jira, DB...) Integrações e automações
Task (subagent) Delega subtarefas a agentes paralelos Pedir ajuda a um colega
Princípio de design: O Claude Code não tem "ferramentas especiais de IA" — ele usa as mesmas ferramentas que um desenvolvedor: grep para buscar, bash para executar, git para versionar. A diferença é que o modelo decide quando e como usar cada uma.

Exemplo prático: a mesma tarefa nos dois paradigmas

Vamos comparar como um chatbot e um agente lidam com a mesma solicitação: "Adicione validação de email no formulário de cadastro."

💬 Abordagem Chatbot

Gera um bloco de código genérico com regex de email e te diz "cole isso no seu formulário". Não sabe qual framework você usa, onde está o form, nem se o código compila.

🤖 Abordagem Agente

Busca o form com Grep, lê o arquivo, identifica o framework (React + Zod por ex.), implementa a validação no padrão do projeto, roda os testes, e corrige se quebrarem.

O que o agente faz por baixo dos panos

Sequência real de tool calls do Claude Code ~8 tool calls 
// 1. Explorar — encontrar o formulário
Grep { pattern: "cadastro|signup|register", glob: "**/*.tsx" }
  → encontra: src/components/SignupForm.tsx

// 2. Ler — entender o contexto
Read { file: "src/components/SignupForm.tsx" }
  → identifica: React, react-hook-form, schema Zod

Read { file: "src/schemas/signup.schema.ts" }
  → identifica: schema Zod existente sem validação de email

// 3. Planejar — (extended thinking)
  → "O projeto já usa Zod. Vou adicionar .email() no schema
     e uma mensagem de erro consistente com as demais."

// 4. Agir — implementar
Edit { file: "src/schemas/signup.schema.ts",
       old: "email: z.string()",
       new: "email: z.string().email('Informe um email válido')" }

// 5. Verificar — rodar testes
Bash { command: "npm test -- --grep signup" }
  → 1 teste falhou: snapshot desatualizado

// 6. Corrigir — o loop de feedback em ação
Bash { command: "npm test -- --grep signup --updateSnapshot" }
  → ✓ Todos os testes passaram

// 7. Resultado entregue ao desenvolvedor para revisão
O ponto-chave: O agente não "sugere código" — ele resolve o problema de ponta a ponta. Leu o codebase, respeitou as convenções do projeto, implementou a mudança e verificou com testes. Tudo dentro de um único prompt do usuário.

Espectro de Autonomia

A diferença entre chatbot e agente não é binária — existe um espectro de autonomia. A Anthropic publicou o paper "Measuring AI Agent Autonomy in Practice" (2025) propondo um framework com cinco níveis, baseados no papel do humano na interação:

Operador
Nível 1

Humano controla tudo. IA só gera texto sob demanda. → Chatbot clássico

Colaborador
Nível 2

IA sugere ações, mas humano aprova cada uma. → Copilot com confirmação

Consultor
Nível 3

IA executa e pede aprovação em decisões críticas. → Claude Code (modo default)

Aprovador
Nível 4

IA executa autonomamente e humano revisa depois. → Claude Code (modo auto-accept)

Observador
Nível 5

IA opera com autonomia total. Humano só monitora. → Headless / CI pipelines

Na prática: O Claude Code no modo default (nível 3) é o ponto de equilíbrio para a maioria dos cenários. Ele pede permissão antes de executar comandos bash ou editar arquivos, mas pode usar ferramentas de leitura (Read, Grep, Glob) livremente. Nos módulos de segurança veremos como controlar esse nível com modos de permissão.

O poder da auto-correção

A capacidade mais importante que separa um agente de um chatbot é a auto-correção. A documentação oficial da Anthropic sobre best practices destaca que agentes que conseguem verificar e melhorar seu próprio output são fundamentalmente mais confiáveis: erros são pegos antes de se acumularem, desvios são corrigidos e a qualidade melhora a cada iteração.

O loop de auto-correção: implementa, verifica, e se falhar volta ao início com mais contexto sobre o erro.

Resumo comparativo

Dimensão 💬 Chatbot 🤖 Agente (Claude Code)
Modo de operação Reativo (responde quando perguntado) Proativo (decide próximos passos)
Acesso ao file system Nenhum Read, Write, Edit, Glob, Grep
Execução de código Não executa Bash (comandos, testes, builds)
Verificação Confia que o código está certo Roda testes, analisa output, itera
Contexto do projeto Só o que o usuário colou no chat Lê o codebase inteiro sob demanda
Loop de feedback 1 turno (pergunta → resposta) N turnos (observar → agir → verificar → repetir)
Integração com tools Nenhuma nativa Git, MCP servers, subagentes
Resultado típico Bloco de código para copiar/colar Código implementado, testado e pronto pra commit

O que isso muda na forma de trabalhar

Entender que o Claude Code é um agente — e não um chatbot — muda a forma como você formula seus prompts e estrutura suas interações. Aqui estão as implicações mais importantes:

🎯 Descreva o objetivo, não os passos

Em vez de ditar cada ação ("abra o arquivo X, mude a linha Y"), descreva o resultado esperado ("adicione validação de email no signup"). O agente descobre o caminho.

🔍 Forneça critérios de verificação

Diga como saber se a tarefa está feita: "todos os testes devem passar", "a build deve compilar", "a API deve retornar 200". Isso alimenta o loop de auto-correção.

📐 Confie no contexto do projeto

Não precisa colar código no chat — o agente pode ler seu codebase. Referencie arquivos por nome ou padrão e ele encontra sozinho.

🔄 Espere iteração, não perfeição instantânea

O agente pode errar no primeiro attempt e se corrigir. Isso é normal e esperado — faz parte do loop agêntico. A primeira tentativa não precisa ser perfeita.

Erro comum: Tratar o Claude Code como chatbot — colando código no prompt e pedindo para "corrigir esse trecho". Em vez disso, aponte para o arquivo e descreva o problema. Deixe o agente usar suas ferramentas para explorar, entender o contexto completo e implementar a solução.
