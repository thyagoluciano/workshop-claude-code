Modulo 3 . Topico 1

Plan Mode: separar pesquisa de execucao

Antes de deixar o agente editar seu codigo, deixe-o entender o problema. O Plan Mode transforma o Claude Code em um explorador read-only que le, busca e analisa sem tocar em nenhum arquivo — e isso muda drasticamente a qualidade do output.

O que e o Plan Mode?

O Plan Mode e um modo de permissao do Claude Code onde o agente pode usar apenas ferramentas de leitura — Read, Grep, Glob — mas nao pode editar arquivos, escrever novos arquivos ou executar comandos bash. E uma fase de exploracao pura: o Claude le seu codebase, responde perguntas e cria planos de implementacao detalhados, sem risco de alteracoes acidentais.

A ideia central vem de uma das melhores praticas da Anthropic: separar pesquisa e planejamento da implementacao para evitar que o agente resolva o problema errado. Quando voce deixa o Claude pular direto para o codigo, ele pode produzir uma solucao sintaticamente correta mas semanticamente errada — porque nao entendeu o contexto completo.

Principio-chave: "Letting Claude jump straight to coding can produce code that solves the wrong problem. Use plan mode to separate exploration from execution."
— Anthropic, Best Practices for Claude Code

Como ativar o Plan Mode

Existem duas formas de entrar no Plan Mode:

1 Durante a sessao: Shift+Tab

Pressione Shift+Tab para alternar entre os modos de permissao. O primeiro toque ativa o auto-accept mode (aceita edicoes automaticamente). O segundo toque ativa o Plan Mode, indicado por ⏸ plan mode on no prompt. Um terceiro toque volta ao modo default.

2 Na inicializacao: flag CLI

Inicie o Claude Code ja em Plan Mode passando a flag de permissao:

Iniciando em Plan Mode via CLI Terminal 
# Iniciar sessao diretamente em plan mode
claude --permission-mode plan

# Equivalente: iniciar e depois alternar com Shift+Tab x2
claude
# [dentro da sessao] Shift+Tab → Shift+Tab → plan mode on

O que o Claude pode e nao pode fazer no Plan Mode

No Plan Mode, o agente opera exclusivamente com ferramentas de leitura. Qualquer tentativa de usar uma ferramenta de escrita ou execucao sera bloqueada automaticamente.

Ferramenta Tipo Plan Mode O que faz
Read Leitura Permitida Le conteudo de arquivos
Grep Leitura Permitida Busca texto/regex dentro de arquivos
Glob Leitura Permitida Busca arquivos por padrao de nome
Edit Escrita Bloqueada Edita arquivos existentes
Write Escrita Bloqueada Cria ou sobrescreve arquivos
Bash Execucao Bloqueada Executa comandos no terminal
MCP Tools Variavel Bloqueada Ferramentas externas (APIs, etc.)
Implicacao pratica: No Plan Mode o Claude nao consegue rodar npm test, git log ou qualquer comando bash. Ele precisa inferir o estado do projeto apenas pela leitura do codigo-fonte. Por isso e importante fornecer um bom CLAUDE.md com contexto sobre a arquitetura e convencoes do projeto.

Por que separar exploracao de execucao?

A separacao entre pesquisa e acao nao e um capricho — e uma estrategia que melhora significativamente os resultados. A documentacao oficial da Anthropic lista tres razoes principais:

! Reduzir erros

Sem exploracao previa, o agente pode editar o arquivo errado, usar o framework errado ou ignorar convencoes do projeto. Plan Mode forca o entendimento antes da acao.

$ Economizar tokens

Um plano validado pelo dev evita ciclos de tentativa-e-erro caros. Retrabalho consome muito mais tokens do que um planejamento inicial cuidadoso.

? Entender antes de agir

Em codebases complexos, as dependencias e convencoes nao sao obvias. O Plan Mode garante que o agente mapeia o terreno antes de comecar a cavar.

Fluxo recomendado: Plan, Validate, Execute

A Anthropic recomenda um workflow de tres fases para tarefas complexas. A chave esta nos passos 1 e 2 — sem eles, o Claude tende a pular direto para o codigo.

Fluxo em tres fases: explorar com ferramentas read-only, validar o plano com o dev, e so entao executar as mudancas.

Quando usar (e quando nao usar) o Plan Mode

O Plan Mode brilha em cenarios de incerteza e complexidade. Mas para tarefas simples e bem definidas, ele adiciona overhead desnecessario. A regra da Anthropic e direta: se voce consegue descrever o diff em uma frase, pule o planejamento.

+ Use Plan Mode quando...

Codebase novo: voce (ou o Claude) nao conhece a estrutura do projeto
Bug complexo: o problema envolve multiplos arquivos ou camadas
Tarefa ambigua: voce nao tem certeza de qual abordagem seguir
Refatoracao grande: mudancas que afetam muitos arquivos simultaneamente
Multiplos arquivos: a mudanca toca em diferentes modulos ou camadas
Codigo desconhecido: voce esta modificando codigo que nao escreveu
vs

- Pule o Plan Mode quando...

O escopo e claro e o fix e pequeno (typo, log line, rename)
Voce consegue descrever o diff em uma frase
Voce conhece bem o arquivo e o contexto e limitado
E uma tarefa repetitiva que voce ja fez antes com sucesso
O CLAUDE.md ja documenta bem as convencoes relevantes

Comparacao: "code first" vs. "explore first"

A diferenca entre pular direto para o codigo e explorar antes e dramatica em tarefas complexas. Veja a comparacao lado a lado:

Dimensao Sem Plan Mode Com Plan Mode
Contexto inicial Apenas o prompt do usuario Prompt + leitura do codebase + dependencias mapeadas
Risco de erro Alto — pode resolver o problema errado Baixo — entendimento validado antes da acao
Convencoes do projeto Pode ignorar ou adivinhar Identifica padroes existentes e os segue
Ciclos de correcao Muitos — tentativa e erro Poucos — plano validado reduz retrabalho
Consumo de tokens Mais tokens (retrabalho caro) Menos tokens totais (acerta mais cedo)
Previsibilidade Surpresas frequentes no output Dev valida o plano antes — sem surpresas
Melhor para Tarefas simples e bem definidas Tarefas complexas, ambiguas ou multi-arquivo

Exemplos de prompts que funcionam bem no Plan Mode

No Plan Mode, voce quer que o Claude explore e proponha, nao que execute. Use prompts que pedem analise, mapeamento e planejamento.

Exploracao de codebase novo Plan Mode 
# Mapeando a arquitetura
Explore este projeto e me explique a arquitetura geral.
Quais sao os principais modulos, como eles se conectam,
e quais padroes de design sao usados?
Planejamento antes de refatoracao Plan Mode 
# Criando plano de implementacao
Preciso migrar o sistema de autenticacao de JWT para
sessions com Redis. Analise o codigo atual de auth,
mapeie todos os pontos que precisam mudar, e crie um
plano detalhado de implementacao passo a passo.
Investigacao de bug complexo Plan Mode 
# Investigando sem alterar
Os testes de integracao do modulo de pagamentos estao
falhando intermitentemente. Investigue o codigo do
modulo, as fixtures de teste e as dependencias.
Me diga o que voce acha que esta causando o problema
e como voce resolveria.
Avaliacao de impacto Plan Mode 
# Analise de impacto antes de mudar
Quero renomear o modelo User para Account em todo
o codebase. Busque todas as referencias, calcule o
impacto da mudanca e liste todos os arquivos que
precisariam ser alterados, na ordem correta.
Dica: Depois que o Claude apresentar o plano no Plan Mode, voce pode refinar com follow-ups como "e se usarmos o padrao X em vez de Y?" ou "considere tambem o impacto nos testes E2E" — tudo ainda em modo read-only. So mude para o modo de execucao quando o plano estiver solido.

Como o Extended Thinking complementa o Plan Mode

O Extended Thinking e o mecanismo que permite ao Claude dedicar mais "tempo de raciocinio" antes de responder. Quando combinado com o Plan Mode, voce tem uma exploracao que e ao mesmo tempo ampla (le muitos arquivos) e profunda (raciocina mais sobre o que leu).

No Claude Code, voce pode acionar diferentes niveis de thinking usando palavras-chave especificas no prompt:

Palavra-chave Nivel de thinking Quando usar
"think" Padrao estendido Tarefas moderadas — bom ponto de partida
"think hard" Orcamento maior Problemas multi-arquivo ou ambiguos
"think harder" Orcamento alto Arquiteturas complexas ou decisoes criticas
"ultrathink" Orcamento maximo Problemas muito complexos que precisam de analise profunda
Combinando Plan Mode com Extended Thinking Plan Mode 
# Exploracao profunda: plan mode + ultrathink
Think harder sobre a arquitetura deste projeto.
Analise os padroes de design, identifique pontos de
acoplamento entre modulos, e proponha um plano de
refatoracao que melhore a testabilidade sem quebrar
a API publica.
Combinacao poderosa: Plan Mode garante que o Claude so le (sem risco de alteracoes). Extended Thinking garante que ele raciocina profundamente sobre o que leu. Juntos, eles produzem planos de implementacao significativamente melhores do que simplesmente pedir ao Claude para "implementar X".

Workflow completo na pratica

Veja como integrar o Plan Mode no seu fluxo de trabalho diario, do inicio ao fim de uma tarefa:

1

Ativar Plan Mode

Inicie com claude --permission-mode plan ou pressione Shift+Tab x2 numa sessao existente.

2

Descrever a tarefa com contexto

Forneca o objetivo e peca para o Claude explorar o codebase. Ex: "Investigue como o sistema de notificacoes funciona e proponha como adicionar suporte a webhooks."

3

Iterar sobre o plano (ainda em Plan Mode)

Revise o plano proposto. Faca perguntas, peca ajustes, solicite que considere edge cases. Tudo sem risco — nada e editado.

4

Aprovar o plano

Quando estiver satisfeito, diga algo como "O plano esta bom. Vou sair do plan mode para voce implementar."

5

Mudar para modo de execucao

Pressione Shift+Tab para voltar ao modo default (ou auto-accept para mais autonomia). O Claude agora pode editar, escrever e executar.

6

Executar o plano

Diga "Implemente o plano que voce propos e rode os testes para verificar." O Claude seguira o plano e usara o loop de auto-correcao.

Anti-pattern: Nao pule os passos 1-3 em tarefas complexas. Ir direto para a execucao sem explorar o codebase e a causa mais comum de implementacoes que resolvem o problema errado. O investimento de 2-3 minutos em planejamento evita 15-20 minutos de retrabalho.
