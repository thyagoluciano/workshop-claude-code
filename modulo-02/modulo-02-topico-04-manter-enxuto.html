Modulo 2 . Topico 4

Mantendo o CLAUDE.md enxuto

A regra de ouro: "Se eu removesse isso, o Claude cometeria erros?" Cada linha do seu CLAUDE.md consome tokens em toda mensagem. Aprenda a manter apenas o que realmente muda o comportamento do modelo.

O principio: cada linha paga aluguel em tokens

O CLAUDE.md e carregado automaticamente no inicio de toda conversa com o Claude Code. Isso significa que cada linha do arquivo consome tokens em cada mensagem que voce envia. Um CLAUDE.md com 500 linhas de instrucoes detalhadas pode parecer completo, mas na pratica ele compete por atencao com o seu codigo e o seu prompt.

A Anthropic recomenda explicitamente manter o CLAUDE.md conciso. A documentacao oficial de best practices orienta que voce pergunte, para cada linha: "Se eu removesse isso, o Claude cometeria erros?" Se a resposta for nao, aquela linha esta desperdicando tokens.

Alerta: CLAUDE.md inchados fazem o Claude ignorar suas instrucoes reais. Quanto mais tokens de contexto sao consumidos por instrucoes genericas, menos "atencao" o modelo tem para o que realmente importa. Isso e o fenomeno de context rot documentado pela Anthropic.

O custo real: tokens consumidos em CADA mensagem

Para entender o impacto, vamos visualizar quantos tokens diferentes tamanhos de CLAUDE.md consomem. Lembre-se: esses tokens sao adicionados ao contexto em toda interacao, nao apenas na primeira mensagem.

Enxuto (50 linhas)
~800 tk

~2.500 tk
Detalhado (300 linhas)

Em uma sessao com 50 mensagens, um CLAUDE.md de 10k tokens adiciona ~500k tokens ao custo total (prompt caching reduz o custo financeiro, mas nao reduz a competicao por atencao no contexto).

Como medir: Use o comando /usage no Claude Code para ver o consumo de tokens da sua sessao. O comando /context mostra exatamente o que esta ocupando espaco no contexto atual. A Anthropic recomenda manter o CLAUDE.md abaixo de 200 linhas.

Tecnicas para manter o CLAUDE.md enxuto

A documentacao oficial da Anthropic trata o CLAUDE.md como um prompt que deve ser refinado iterativamente. Aqui estao as tecnicas mais efetivas para mante-lo conciso:

1 Bullets em vez de paragrafos

Uma instrucao por linha, sem texto explicativo. O Claude entende perfeitamente - Use pnpm, nao npm sem precisar de um paragrafo justificando.

2 Referencie, nao cole

Em vez de colar snippets inteiros, aponte para o arquivo: - Siga o padrao de src/utils/api-client.ts. O Claude pode ler o arquivo sob demanda.

3 Mova instrucoes especificas para rules/

Instrucoes que so se aplicam a certos tipos de arquivo pertencem a .claude/rules/, nao ao CLAUDE.md global. Rules sao carregadas sob demanda, nao em toda sessao.

4 Mova workflows para Skills

Instrucoes detalhadas sobre processos especificos (review de PR, migracoes, deploy) devem virar Skills. Skills sao carregadas on-demand, nao em toda sessao.

5 Use enfase com criterio

A Anthropic usa IMPORTANT e YOU MUST para melhorar aderencia, mas se tudo e "IMPORTANT", nada e. Reserve para 2-3 regras criticas.

6 Remova o obvio

Nao instrua o Claude a "escrever codigo limpo" ou "seguir boas praticas" -- ele ja faz isso. So documente comportamentos que diferem do padrao.

Dica da Anthropic: Trate o CLAUDE.md como um prompt de producao. A equipe da Anthropic relata que ocasionalmente passa o CLAUDE.md pelo prompt improver e ajusta instrucoes adicionando enfase para melhorar a aderencia. Itere sobre o CLAUDE.md como voce iteraria sobre qualquer prompt critico.

Fluxo de manutencao: o ciclo de vida de uma regra

O CLAUDE.md e documentacao viva -- deve evoluir junto com o projeto. A Anthropic descreve um loop de melhoria continua onde o Claude Code ajuda a refinar as proprias instrucoes do CLAUDE.md com base no uso real.

1

Observe um erro recorrente

O Claude insiste em usar npm quando seu projeto usa pnpm, ou gera testes com Jest quando voce usa Vitest.

2

Adicione uma regra concisa

Adicione ao CLAUDE.md: - IMPORTANT: Use pnpm, nunca npm. Mantenha como bullet point, uma linha, sem explicacao longa.

3

Teste em sessoes reais

Nas proximas sessoes, observe se o Claude agora acerta. Se sim, a regra esta pagando seu aluguel. Se nao, ajuste a redacao ou adicione enfase.

4

Revise periodicamente

A cada sprint ou mensalmente, revise o CLAUDE.md inteiro. Remova regras que o modelo ja segue por padrao, instrucoes para features removidas, ou duplicacoes.

5

Promova ou rebaixe

Regras especificas demais vao para .claude/rules/. Workflows detalhados viram Skills. So o essencial fica no CLAUDE.md raiz.

Exemplo pratico: refatorando um CLAUDE.md

Veja como transformar um CLAUDE.md inchado em uma versao enxuta e efetiva, aplicando as tecnicas acima. O objetivo e manter apenas instrucoes que mudam o comportamento do Claude.

ANTES (~95 linhas, ~1.500 tokens)

# Reducao: ~80% menos tokens
# Tudo que foi removido: o Claude ja
# sabe ou pode inferir do codigo

O que foi removido e por que

Conteudo removido Motivo
Descricao do projeto O Claude infere isso do package.json e da estrutura de pastas
Lista de tecnologias Visivel no package.json, tsconfig.json e imports
"Escreva codigo limpo" Comportamento padrao do modelo -- nao agrega valor
Como rodar o projeto Redundante com scripts no package.json (exceto pnpm vs npm)
Estrutura de pastas O Claude usa Glob e Read para explorar a estrutura
Paragrafos explicativos Substituidos por bullets concisos -- mesma informacao, menos tokens
Resultado: De ~1.500 tokens para ~300 tokens -- uma reducao de 80%. Em 50 mensagens, isso economiza ~60.000 tokens de contexto. Mais importante: as instrucoes que restaram tem maior chance de serem seguidas, porque o modelo nao precisa competir com "ruido".

Onde colocar cada tipo de instrucao

O Claude Code oferece multiplos locais para armazenar instrucoes. A chave e colocar cada tipo de informacao no local certo para minimizar o consumo de tokens do CLAUDE.md raiz.

Tipo de instrucao Onde colocar Carregamento
Comandos de build/test/lint CLAUDE.md (raiz) Toda sessao
Convencoes globais do projeto CLAUDE.md (raiz) Toda sessao
Preferencias pessoais CLAUDE.local.md Toda sessao (so sua)
Regras para tipo de arquivo .claude/rules/*.md Sob demanda (por path)
Workflows (PR review, deploy) .claude/skills/*.md Sob demanda (invocado)
Preferencias globais (todos os projetos) ~/.claude/CLAUDE.md Toda sessao
Regras de subpastas (monorepo) packages/*/CLAUDE.md Sob demanda (por path)
Principio-chave: Quanto mais especifica a instrucao, mais fundo na hierarquia ela deve ficar. O CLAUDE.md raiz e "caro" em tokens -- so o que e verdadeiramente global deve estar la. A documentacao oficial recomenda manter o CLAUDE.md abaixo de 200 linhas movendo instrucoes especializadas para Skills.

Como calcular o custo do seu CLAUDE.md

Voce pode estimar quantos tokens seu CLAUDE.md consome usando ferramentas simples. A regra geral e: ~1 token para cada 4 caracteres em ingles, ou ~1 token para cada 3 caracteres em portugues.

Estimando tokens do seu CLAUDE.md Terminal 
# Contagem rapida de caracteres e estimativa de tokens
$ wc -c CLAUDE.md
1847 CLAUDE.md

# Estimativa: ~1847 / 3 = ~616 tokens (portugues)
# Estimativa: ~1847 / 4 = ~462 tokens (ingles)

# Contagem de linhas (meta: < 200)
$ wc -l CLAUDE.md
47 CLAUDE.md

# No Claude Code, use /usage para ver o consumo real
# e /context para ver o que esta no contexto atual

$ Custo por sessao

Um CLAUDE.md de 500 tokens a $3/MTok (input cacheado) custa ~$0.0015 por mensagem. Em 50 msgs: ~$0.075 por sessao.

$ Custo por mes

Com 10 sessoes/dia, 20 dias/mes: ~$15/mes so do CLAUDE.md. Um CLAUDE.md enxuto reduz para ~$3/mes.

! Custo real

O custo financeiro e secundario. O custo real e a perda de atencao: instrucoes ignoradas porque o contexto esta poluido.

CLAUDE.md como documentacao viva

O CLAUDE.md nao e um documento estatico -- e uma ferramenta de calibracao que deve evoluir junto com o projeto. A Anthropic descreve um loop de melhoria onde o Claude Code ajuda a refinar suas proprias instrucoes com base no uso real.

+ Quando adicionar

O Claude comete o mesmo erro mais de uma vez; uma convencao do projeto nao pode ser inferida do codigo; um comando de build/test nao e obvio.

- Quando remover

O modelo ja segue a regra por padrao; a feature foi removida; a instrucao e redundante com outra; a regra nunca mudou o comportamento do Claude.

Anti-pattern: Muitas equipes tratam o CLAUDE.md como um "README para o Claude" -- documentando tudo sobre o projeto de forma exaustiva. Mas o Claude ja sabe explorar o codebase com suas ferramentas. O CLAUDE.md deve conter apenas o que o Claude nao consegue descobrir sozinho.

Resumo: checklist do CLAUDE.md enxuto

  • 1. Para cada linha, pergunte: "Removendo isso, o Claude erraria?"
  • 2. Mantenha abaixo de 200 linhas (recomendacao oficial da Anthropic)
  • 3. Use bullet points, nao paragrafos explicativos
  • 4. Referencie arquivos em vez de colar codigo
  • 5. Mova instrucoes especificas para .claude/rules/
  • 6. Mova workflows detalhados para Skills
  • 7. Reserve IMPORTANT e YOU MUST para 2-3 regras criticas
  • 8. Nao documente o que o Claude ja infere do codigo (stack, estrutura, etc.)
  • 9. Revise periodicamente e remova regras obsoletas
  • 10. Use /usage e /context para monitorar o consumo
